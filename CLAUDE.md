# KicksWrap 発注自動化エージェント

このリポジトリは、KicksWrapの在庫管理・発注業務を自動化するためのプロジェクトです。
「在庫が減ったら、ルールに従って発注の下書きを自動で作ってくれる仕組み」を構築します。

---

## このリポジトリの使い方

Claude Code を開いた状態で、以下のコマンドを入力してください。
上から順番にやっていくのがおすすめです。

| コマンド | 何をするか | いつ使う |
|---------|-----------|---------|
| `/setup` | 環境の確認と初期設定 | 最初に1回だけ |
| `/build-knowledge` | 発注ルール・商品知識を言語化 | ナレッジを書くとき（何回でもOK） |
| `/dev` | GASの実装をガイド | コードを書くとき |

### コマンドの使い方

1. このフォルダでClaude Codeを起動（ターミナルで `cd` してから `claude`）
2. チャットに `/setup` と入力してEnter
3. あとは対話で進む。途中でやめても、次回 `/build-knowledge` で続きから再開できる

## ファイル構成

```
kickswrap-ordering/
│
├── CLAUDE.md              ← 今読んでるこのファイル
│
├── 発注者.md               ← ★ 一番大事。発注ルール・判断基準を書く場所
│                             最初は空欄付きテンプレート。/build-knowledge で埋めていく
│
├── products/              ← 商品ごとの個別ルール（1商品1ファイル）
│   └── _template.md         商品ルールのテンプレート（コピーして使う）
│
├── templates/
│   └── hanchu.md          ← ヲヲフェニックスへの発注依頼文のひな形
│
├── gas/                   ← Google Apps Script のコード
│   ├── README.md            GASエディタへの貼り付け方の説明
│   ├── Code.gs              メインスクリプト（在庫チェック・通知）
│   └── Config.gs            設定値（スプシID・Slack URL等）
│
└── .claude/commands/      ← Claude Codeのナビゲーションスキル（裏方）
    ├── setup.md              環境確認
    ├── build-knowledge.md    ナレッジ構築ナビゲーター
    └── dev.md                GAS開発ガイド
```

---

## 開発の全体像

```
Step 1: /setup              環境OK？を確認
            ↓
Step 2: /build-knowledge    発注者.md + 商品ルールを対話で言語化
            ↓
Step 3: /dev                GASを実装（在庫チェック → Slack通知 → 定期実行）
            ↓
Step 4: Claude API 追加      発注者.mdを読んで自動判断させる（APIキー発行後）
```

Step 4 は後から追加する設計なので、先に Step 1〜3 だけ進めてOK。

---

## ルール

- 発注の最終判断は必ず人間がやる（全自動にはしない）
- APIキー・パスワード等はこのリポジトリに直接書かない（Config.gs に入れてGit管理外にする）
- 発注者.md は「育てるファイル」。運用しながら追記・修正していく
